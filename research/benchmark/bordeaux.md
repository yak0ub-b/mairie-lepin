# Bordeaux.fr — Analyse design

## Navigation
- 6 items : Accueil / Le Mag / Vivre à Bordeaux / Sortir / La Mairie / Grands projets
- Multi-niveaux profonds (jusqu'à 4 niveaux de sous-menus)
- Barre de recherche proéminente avec suggestions fréquentes

## Hero
- Carousel rotatif (actualités + événements + annonces)
- Format : image 16:9 + titre + date + catégorie
- **CTA absent** sur le hero — pas de bouton d'action primaire

## Couleurs
- **Primaire** : `#342fff` (bleu vif) — variable CSS `--reading-progress-bar--color`
- Fond blanc standard
- Texte noir/gris neutres
- Accents : gris pour contenus secondaires

## Typographies
- Aucune Google Font détectée — font-stack système probable

## Sections homepage (ordre)
1. Nav + barre de recherche
2. **Accès rapide** (5 liens : mairie, cantines, piscines, stationnement, état civil)
3. Carousel actualité (4 articles épinglés)
4. "En ce moment" (4 articles 3-2)
5. "En bref" (3 brèves)
6. "Le Mag" avec CTA
7. Agenda (850+ événements)
8. Dossier thématique
9. Services et infos pratiques (6 blocs)
10. Newsletters + réseaux sociaux
11. Footer : coordonnées, liens, mentions légales

## Features notables
- ✅ Recherche proéminente avec suggestions
- ✅ Accessibilité : skip-link, ARIA, service Acce-o (malentendants)
- ✅ RGPD : Tarteaucitron + CNIL
- ✅ Réseaux sociaux : 5 canaux (FB, Insta, YouTube, LinkedIn, Bluesky)
- ❌ Pas de multilingue
- ❌ Pas de chatbot

## Points faibles
- Hero sans CTA → appel à l'action diffus
- Trop de niveaux de sous-menus (navigation confuse sur mobile)
- Palette minimaliste → risque monotonie (bleu + blanc + gris)
- Carrousel sans indication de progression
- Section services sans icônes (texte seul)

## Note globale
7/10 — L'une des meilleures structures : accès rapides en position 2 (excellent), recherche proéminente, riche en contenu. Design trop neutre mais organisation exemplaire.
