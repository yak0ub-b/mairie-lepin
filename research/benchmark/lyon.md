# Lyon.fr — Analyse design

## Navigation
- 4 items principaux : Démarches / Sortir et découvrir / Vie pratique / Actions et projets
- Sous-menu d'arrondissements (9 mairies locales)
- Barre de recherche intégrée

## Hero
- Carousel rotatif (3-5 slides) d'actualités
- CTA : liens texte "Lire" — pas de bouton primaire fort
- Contenu récent : élections, conseils municipaux, événements

## Couleurs
- Bleu dominant (logo, marqueurs carte) — hex non exposé dans CSS inline
- Fond blanc, texte noir/gris
- Palette très sobre, peu distinctive

## Typographies
- Non détectées dans le HTML (styles externalisés)
- Stack système probable

## Sections homepage (ordre)
1. Header + nav
2. Carousel actualités
3. "On en parle" (3 articles)
4. "En bref" (4 brèves datées)
5. Accès rapides (8 raccourcis)
6. Grands rendez-vous (carousel événements)
7. Publications à télécharger
8. Focus thématique (3 projets)
9. Newsletter + footer

## Features notables
- ✅ Recherche intégrée (nav + footer)
- ✅ Accessibilité RGAA (partiellement conforme)
- ✅ Analytics Matomo
- ✅ Géolocalisation (9 arrondissements)
- ❌ Pas de chatbot
- ❌ Monolangue

## Points faibles
- Trop de sections de même poids visuel → lecture difficile
- 3+ carousels sur la même page → UX confuse
- Palette fade, peu d'identité visuelle forte
- Densité : 40+ éléments cliquables above the fold

## Note globale
6/10 — Structure solide, bon contenu, mais design trop générique et surchargé. Pas d'identité visuelle forte.
